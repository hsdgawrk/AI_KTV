# Room and Vocal Volume Controls

**Accompaniment Volume** is a room-level 0-100 integer setting that starts at 70, any connected **Paired Slave** may adjust, and it persists across **Current Song** changes; setting it to 0 mutes the base audio without pausing playback or **Playback Progress**. **Vocal Volume** is a per-**Slave** 0-100 integer setting that starts at 70 for a new **Paired Slave**, only the corresponding connected **Paired Slave** may adjust for its own **Vocal Input**, and it persists across songs and reconnection for the current **Paired Slave** lifecycle without becoming a long-lived profile setting. Setting **Vocal Volume** to 0 does not change **Vocal Input State**, and **Vocal Volume** may be adjusted even while **Vocal Input Availability** is unavailable or interrupted; participation in **Mixed Audio** remains controlled by **Vocal Input State**. **Master** may show room and per-slave volume values as read-only status but is not a volume control surface in the first implementation. Clients should throttle slider updates while guaranteeing the final value is sent, while the **Server** simply applies each valid command, clamps and rounds incoming values because clients are not trusted, and keeps per-song asset loudness correction outside the room volume model.
