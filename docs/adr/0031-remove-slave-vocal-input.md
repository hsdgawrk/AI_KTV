# Remove Slave Vocal Input

**Slave** will no longer capture or send live singing audio, and the **Server**, **Master**, and **Slave** runtime protocol will not carry WebRTC signalling, **Vocal Input** availability, **Vocal Input** state, or per-slave vocal volume. **Slave** remains the point-song and room-control role, while karaoke audio now comes only from **Original Vocal** or **Accompaniment** song assets prepared through **Song Import**.
