# Changelog

## v1.1.0 — 2025-07-25

### 🚀 性能优化

- **NCM 解密速度提升 ~50x**：音频解密从逐字节 Python 循环改为 `strxor` 整块 XOR（C 实现），1MB chunk，60MB FLAC 解密耗时从 ~3s 降至 ~65ms

### 🛡️ 可靠性 & 数据安全

- **输出文件原子写入**：先写 `.tmp` 临时文件，完成后 `replace()` 到最终路径，断电/崩溃不会产生损坏的半成品文件
- **配置文件原子写入**：`config.json` 同样采用 write-tmp + replace 策略
- **磁盘满保护**：写入后校验文件大小，磁盘空间不足时立即报错而非生成损坏文件
- **PKCS#7 填充验证**：解密时验证所有填充字节一致性，防止损坏数据静默通过

### 🐛 Bug 修复

- **下拉框点击收回**：MP3 比特率/转换成功后的下拉菜单，再次点击可收回（之前只能点击外部关闭）
- **存量文件扫描不再白等**：启动时扫描已有 NCM 文件跳过 stable-wait（mtime > 5s 直接处理），启动速度大幅提升
- **force_mp3 冗余标签写入**：移除对临时 FLAC 中间文件的无用 `_embed_flac_tags` 调用
- **PyInstaller 打包后 ffmpeg 路径**：打包为 exe 后正确从 exe 同级目录定位 ffmpeg

### 🔒 线程安全

- **Config 跨线程保护**：`convert_one` 入口快照配置字段，隔离 GUI 线程中途修改
- **Watcher start/stop 加锁**：托盘回调跨线程操作 watcher 不再有竞态风险

### 💅 UI 改进

- **转换成功日志显示完整输出路径**：日志面板显示 `→ D:\Music\歌名.mp3` 而非仅文件名
- **资源管理器自动刷新**：转换完成后通知 Windows Shell，输出文件夹无需手动 F5 即可看到新文件
